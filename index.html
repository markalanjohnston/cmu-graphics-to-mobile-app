Refactoring a Python Graphics Project into a Mobile App

Students translate a CMU CS Academy Python project into a working React Native mobile app — with help from AI. A hands-on lesson in concept transfer, debugging, and cross-platform thinking.

Overview

Students take an existing interactive program built in Python using the CMU CS Academy graphics library and refactor it into a mobile application using CodeHS React Native.

The goal is not to memorize new syntax — it is to recognize that the same programming concepts exist across every language and platform. Events, state, conditionals, and drawing logic all transfer. Only the framework changes.

Live demo

The finished app is publicly viewable at codehs.com/sandbox/id/mickeyscreen-AtogQW . Open in the Expo Go app on a tablet or phone to run it on a real device.

Why This Activity Works

By refactoring an existing project, students focus on understanding logic rather than inventing an idea from scratch. Every CMU Graphics concept has a direct React Native equivalent:

Classroom Implementation

Course: Fundamentals of Computer Science    Grade: 9–10

1

Explore the Original Program

Students analyze the CMU Graphics project and identify variables, events, state, and drawing logic.

2

Plan the Mobile Version

Students sketch a mobile layout and decide how touch replaces mouse interaction.

3

Refactor with AI

Students paste their Python code into Claude.ai, then debug the output iteratively in CodeHS.

4

Reflect

Students discuss what transferred, what the AI got wrong, and what decisions required human judgment.

Learning Objectives

  • Recognize how programming concepts translate across languages and platforms
  • Practice debugging and iterative development in a real environment
  • Build and run a basic mobile application
  • Collaborate effectively with an AI coding assistant
  • Communicate technical problems clearly and precisely
  • Reflect on how design decisions change across platforms
